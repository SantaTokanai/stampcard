# スタンプカードアプリ 運用マニュアル（備忘録）

最終更新：2026年7月

---

## 1. このアプリの構成

| 画面 | ファイル | 用途 | ログイン方法 |
|---|---|---|---|
| 会員ページ | `index.html` | スタンプ・ポイント・グッズ交換など | ニックネーム＋パスワード |
| 管理者ページ | `admin.html` | 誰が何を申し込んだか確認、配送用URL入力 | 管理者パスワードのみ |

- 管理者ページは**どこからもリンクしていない**。URLを直接ブックマークしておくこと。
- コードは GitHub（`SantaTokanai/stampcard`）で管理。実際のサイトは GitHub Pages で自動公開される。
- データベースと裏側の処理（ログイン確認・ポイント計算など）は Firebase（プロジェクト名：`stampcard-project`）。

### よく使うURL
- GitHubリポジトリ：`https://github.com/SantaTokanai/stampcard`
- Firebaseコンソール：`https://console.firebase.google.com/project/stampcard-project`
- 会員ページ（本番）：GitHub → Settings → Pages で確認できる
- 管理者ページ：会員ページのURLの末尾に `admin.html` をつけたもの

## 2. 「コードを書き換えなくてもできる」日常操作

このセクションの操作は、**Cloud Shellもコードの変更も不要**。Firebaseコンソールと管理者ページだけで完結する。

### 2-1. 半年に一度、新しい交換会を作る

1. Firebaseコンソール →「Firestore Database」→ `exchange_events` コレクションを開く
2. 「＋ドキュメントを追加」
3. ドキュメントIDは半角で `2026-07` のように「年-月」で入力
4. 以下のフィールドを追加：

   | フィールド名 | 型 | 内容 |
   |---|---|---|
   | `title` | string | 表示名。例：`2026年7月交換会` |
   | `status` | string | 下記2-2を参照 |
   | `items` | array | グッズ一覧。要素は map 型で `name`（string）と `cost`（number）を持たせる。画像を付けたい場合は `img`（string、例：`images/goods/○○.png`）も追加 |

### 2-2. 交換会の状態（`status`）を切り替える

`exchange_events` の該当ドキュメントを開き、`status` フィールドの値を書き換える。

| 値 | 意味 |
|---|---|
| `preview` | グッズは見えるが「決定」ボタンは押せない（下見期間） |
| `open` | 「決定」ボタンが押せる（受付中） |
| `closed` | セクションごと非表示になる（終了・履歴行き） |

**運用の流れの目安**：`preview`（下見期間を設ける）→ `open`（受付）→ `closed`（終了）

### 2-3. 誰が何を申し込んだか確認する

1. 管理者ページ（`admin.html`）を開く
2. 管理者パスワードを入力してログイン
3. 上部のプルダウンで確認したい交換会を選ぶ
4. 申し込み一覧（ニックネーム・内容・消費pt・日時）が表示される

### 2-4. 匿名配送用のURLを伝える

1. 管理者ページで、該当ユーザーの申し込みの下にある「配送用URLを貼り付け」欄にURLを入力
2. 「保存」ボタンを押す
3. 会員本人がマイページを開くと、そのグッズ交換欄に「📦 配送用URLはこちら」ボタンが表示される（本人以外には見えない）
4. URLを消したい場合は、欄を空にして再度「保存」

### 2-5. 合言葉（スタンプ）を新しく追加する

1. Firebaseコンソール →「Firestore Database」→ `keywords` コレクション
2. 「＋ドキュメントを追加」、ドキュメントIDが合言葉そのものになる
3. フィールド：`img`（画像パス）、`x`・`y`（スタンプ位置、0〜1の割合）、`widthPercent`（スタンプの大きさ）
4. スタンプ画像は事前にGitHubの `images` フォルダにアップロードしておく

## 3. 「コードの変更・デプロイが必要」な操作

見た目のデザイン変更や、新機能の追加は、コードの修正と反映が必要。**基本的には生成AI（Claude等）に相談しながら進めるのが安全。**

### 3-1. 画面のファイル（`index.html`, `style.css`, `script.js`, `admin.html`, `admin.js`, `admin.css`）を直す場合
1. GitHubで対象ファイルを開き、鉛筆マーク（Edit）
2. 内容を書き換えて「Commit changes」で保存
3. 1〜2分で自動的にサイトに反映される（GitHub Pages）

### 3-2. 裏側の処理（Cloud Functions／`functions/index.js`）を直す場合
Cloud Shellというブラウザ上のツールを使う。**既存の関数を巻き込まないよう、必ず変更した関数名だけを指定してデプロイすること。**

1. `https://console.cloud.google.com/?project=stampcard-project` を開く
2. 右上の「＞_」アイコンでCloud Shellを開く
3. 初回のみ：`gcloud config set project stampcard-project`
4. `cd ~/stampcard-functions`
5. エディタ（鉛筆マーク）で `functions/index.js` を開き、内容を書き換えて保存
6. ターミナルで以下を実行（`関数名` の部分は実際に変更した関数名に置き換える）：
   ```
   firebase deploy --only functions:関数名
   ```
7. 確認画面で、**変更した関数名だけが表示されていること**を必ず確認してから進める

### 3-3. 管理者パスワードを変更したい場合
1. 3-2の手順で `functions/index.js` を開く
2. ファイル内の `ADMIN_PASSWORD` の値を新しいパスワードに書き換える
3. 保存後、以下を実行：
   ```
   firebase deploy --only functions:adminGetSubmissions,functions:adminSetShippingUrl
   ```
4. 新しいパスワードは、誰にも共有せず自分だけが分かる場所に控えておく

## 4. 困ったときの確認ポイント

### 4-1. ボタンが反応しない・画面が真っ白
- ブラウザで右クリック →「検証」→「Console」タブに赤いエラーが出ていないか確認
- 直前にコードを編集した場合、余分な文字（コピペミス）が紛れていないか確認
- ブラウザの強制再読み込み（Mac: `Cmd+Shift+R` / Windows: `Ctrl+Shift+R`）を試す

### 4-2. 「索引（インデックス）が必要です」というエラーが出た場合
- Firestoreに新しい種類の検索を行う機能を追加した直後に発生することがある
- エラーメッセージ内のリンクをクリックし、「作成」ボタンを押す
- 数分待って「有効」になれば解消

### 4-3. 費用が心配なとき
- `https://console.firebase.google.com/project/stampcard-project/usage` で今の使用量・費用を確認できる
- この規模のアプリでは、通常は無料枠内〜月数円程度で収まる

## 5. データベース（Firestore）のコレクション一覧

| コレクション名 | 内容 | 手動で編集してよいか |
|---|---|---|
| `users` | 会員情報（ニックネーム・パスワード・スタンプ・ポイント等） | 基本は触らない（ポイント調整など緊急時のみ慎重に） |
| `keywords` | スタンプの合言葉・画像・位置情報 | ○ 新規追加してよい |
| `requests` | 曲リクエスト | 基本は触らない |
| `exchange_events` | 交換会の情報（グッズ一覧・状態） | ○ 新規作成・`status`変更してよい |
| `exchange_submissions` | 会員の申し込み内容・配送用URL | 配送用URLの確認以外は触らない（自動生成される） |

---

## 6. 後片付けを忘れずに（テスト作業をした後）

新機能をテストした後は、以下を忘れずに行う。

- テストで作った `exchange_submissions` のドキュメントを削除
- テストアカウントの `users` の `spentPoint` を0に戻す
- `exchange_events` の `status` を本番用の状態（`preview`または`closed`）に戻す
- 配送用URLのテスト値を空にして保存し直す
